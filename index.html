¿Qué significa CRUD?

Es el acrónimo de Create Read Update y Delete.

Las ventajas que ofrece son:

  • Facilita la creación y gestión de datos.
  • Proporciona una estructura coherente y fácil de entender para su manipulación.
  • Ayuda a minimizar los errores y garantiza la integridad de los datos.
  • Proporciona una base sólida para el desarrollo de aplicaciones.

Por otro lado, lo utilizaría para comprender o explicar las cuatro operaciones básicas que pueden realizarse en una base de datos y lo utilizaría a la hora de diseñar el comportamiento de una apliación que vaya a requerir de estos elementos. La operación Create sirve para crear un nuevo elemento, la operación Read para obtener datos de uno o varios elementos de una base de datos, Update para actualizar o modificar los datos de un elementos en la base de datos y por último Delete para borrar un elemento o datos concretos de ese elemento.

Por ejemplo en una aplicación que muestre nuestro portfolio, podemos crear un panel de control donde se puedan añadir nuevos proyectos, obtener los datos de los proyectos realizados para mostrarlos, actualizar fotos, títulos, descripciones u otros elementos del proyecto y por último borrar algunos de estos datos o un proyecto completo que no queramos mostrar más.

Otro ejemplo podría ser una eCommerce de ropa. Utilizariamos las operaciones básicas descritas para publicar nuevos productos(Create), mostrar el catálogo de productos(Read), actualizar información de los productos como el stock disponble, precio, fotos, etc. (Update) y borrar artículos como podrían ser los descatalogados(Delete).

Un tercer ejemplo podría ser la gestión de una base de datos de los usuarios registrados en un gimnasio que tenga una aplicación Web. Desde el punto de vista de los usuarios, podrían darse de alta a través de la app y gestionar sus datos. En este caso Create se utilizaría para a añadir un nuevo usuario, Read para obtener los datos del cliente y mostrarlos en su cuenta, Update para modificar sus datos y Delete para borrar información del perfil del usuario o borrarlo por completo en caso de que elimine su suscripción. Desde la perspectiva del desarrollador frontend permite diseñar la estructura del código y ver que tipo de acciones(CRUD) se deben solicitar a la base de datos en función de lo que el usuario demande.

¿Qué es un operador ternario?

Es una variante en la sintaxis de los condicionales para escribirlos en menos líneas de código.

Lo utilizaría para hacer el código más compacto por ejemplo al escribir JSX en React ya que hace más legible y simple el código.

La sintaxis es:

condicion ? (se ejecuta esto si es true) : (se ejecuta esto si es false)

Ejemplo de operador ternario en JavaScript:

true ? console.log(“ejecuta este código”) : console.log(“ejecuta este otro código”)

Resultado: ejecuta este códgio

false ? “ejecuta este código” : “ejecuta este otro código”

Resultado: ejecuta este otro código

¿Qué tipo de solicitud de API hacemos cuando queremos eliminar datos?

Se hace una solicitud Delete mediante la conexión a la api y le pasamos los datos necesarios para que encuentre el elemento o elementos que queremos eliminar.

Por ejemplo, en el caso de un usuario registrado en un gimansio, podría accionarse la petición de eliminar su foto de perfil. Para ello se haría una petición Delete a la api con la referencia o id del cliente para encontrar el apartado donde se guarda la foto del perifl y así eliminarla de la base de datos.

En el ejemplo de la App del porfolio podriamos eliminar un proyecto completo y hariamos una petición Delete a la api de todos los datos del proyecto asociados a su id.

¿Qué tipo de solicitud de API hacemos cuando queremos actualizar los datos?

El tipo de solicitud que realizamos para actualizar los datos es Patch. La ventaja de este tipo de solicitud es que se pueden actualizar sólo los datos que configuremos en la petición.

Lo utilizaría por ejemplo para cambiar la dirección de un cliente en caso de que éste haya cambiado de ubicación.

Otro ejemplo podría ser en una red social que un usuario modificase su foto de perfil. Dicha acción podría ejecutarse a través de una petición Patch.

¿Qué es el código dinámico?

El código dinámico es aquel que permite mostrar contenido dinámico en una aplicación Web y que cambia con las entradas u acciones que realizan los usuarios. Esto ofrece una experiencia de usuario más atractiva y un contenido actualizado.

Por ejemplo, en el caso de un eCommerce el cliente puede acceder a los productos disponibles en existencias gracias a la posibilidad de utilizar un códgio dinámico. También si el usuario aplica algún tipo de filtro como precio, color, tamaño, etc. permite que la página se actualice y muestre sólo esos artículos.

La sintaxis en JavaScirpt para utilizar código dinámico en un cadena se compone de las comillas para atrás y el símbolo “$” seguido de las llaves “{}”. Por ejemplo:

let nombre = “Josu”

console.log(`Mi nombre es ${nombre}.`)

Restultado: Mi nombre es Josu.

El dato que contiene la variable nombre podría obtenerse cuando un usuario esté logeado en su cuenta, mediante una conexión API y que mostrase el nombre correspondiente a cada usuario.
